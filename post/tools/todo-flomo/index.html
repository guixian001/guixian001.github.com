使用 TODO List 结合轻笔记进行知识积累

TODO List 与笔记

使用 TODO List 类的工具可以较好的管理我们的计划事项，但是也会存在一个问题。当欧美原先做过类似的某一项工作，记不起来当时做了什么，有什么积累。在工具上也只有标题和完成状态，对于其他实践经验，有可能都记不起来了。

这意味着虽然我们完成了任务，但是探索的过程却尚未完成，许多经验都需要尽快记下来，留待后续思考。如果我们以「完成任务」为标准，而不去记录这些过程中的思考，就会丢掉许多这样的实践经验。

那么定期做任务复盘不就好了？其实未必，因为任务复盘往往是在任务结束之后，周期太长，许多事情印象都会变得模糊，而复盘往往又过于正式，会丢失掉许多具体的细节。

所以这时候我们就需要再完成任务的同时将任务的实践经验进行记录，比如 flomo。或者其他笔记类的软件均可。记录最好是经过自己的理解消化的知识，而不仅仅的摘录，这样有助于后续自己进行回顾消化。

拆解任务

「拆分」任务是任务管理中非常重要的步骤，在拆解的时候多花精力，才能在执行的时候降低压力。

我们习惯于随手记录任务，然后却发现久久无法开始。缺少时间固然是一方面，而另一方面则是缺少对应的经验。如果无法拆分下去，尽量把任务的范围缩小，毕竟我们拆解任务是为了完成的，不是为了挂在那里向别人展示的。

拆解拆分方式可以源自自己的经验，而非唯一标准。而只有当我们深入了解一件事情的时候，才能把任务拆解到足够细。

这么做有什么好处？一方面能让我们准确地评估任务的结果和耗费的资源，另一方面能让我们在执行任务的过程中不用再去思考问题，可以像激光一样专注于执行。

拆解任务的时候，滴答清单的子任务功能能很好地助你一臂之力。你可以把一个模糊的任务不断精确，拆成小且明确的任务，并且通过共享功能指派给对应的人，提高任务的执行效率。

注意，拆到什么程度算 ok 呢？有个简单的衡量标准：当你通过滴答清单的共享功能，分享给同事时，他们能直接根据任务描述去执行，不需要再找你提问。

当我们拆解完任务，设定好起止时间，利用滴答清单的日历视图，就能快速知道接下来一段时间的重点和进展，就不会再有「疲于奔命」的感觉了。

总结

在使用 TODO List 过程中需要将任务的实践经验进行积累，而不知识关注任务的状态，只有经过积累了后续自己才可以更快的回顾起来。积累可以寻找自己偏好的方式，容易记录并且寻找即可。

在开展任务时需要进行拆分，当我们在推进一项比较大的任务时，很有可能因为没有将任务拆解到足够细从而使得任务无法推进和跟踪状态。没有拆解的任务再进行每次进行时会带来认知负担（我要做什么、做到哪里了、还需要做什么等等）。把任务拆解成一项项具体的行动，会更加具体和推进。
